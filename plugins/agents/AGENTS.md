# AGENTS.md

Brief description of the purpose of the repository and the role of agents (for example, "This repository contains the backend of the SaaS platform. Agents help maintain architectural consistency, document code, and create automations.")

## 2. Agent Principles (Universal Rules)
Universal rules that apply to **all agents** interacting with this repo.

## 3. Interaction Protocol
Rules about **how agents should act** in case of ambiguities, conflicts, or new instructions.

## 4. Repository-Specific Context
Description of the project type, language, framework, and key conventions (naming, architecture, testing, etc.)

## 5. Agent Roles
List of active agents and their responsibilities.
(Example: `doc-agent`, `infra-agent`, `code-review-agent`, `release-agent`, `ai-doc-agent`...)

## 6. Communication Style
Rules about language, tone, format of comments, and commits generated by the agents.

## 7. Exception Management
Procedure for when something contradicts the `AGENTS.md`.

## 8. Update & Governance
How and who can modify this file, and how to document changes.

## 9. Examples (Optional)
Examples of desired or undesired interactions with the agents.
